# 実施76〜84の結果

- **実施76**：2台目のC6実機（ボードC）導入，reflash不要の恒久2台構成（ボードB=ASP3，ボードC=NuttX常時稼働）確立。phy_init呼出し列を最新ビルドで再検証——0/186 IDミスマッチ。
- **実施77**：regi2cアナログ較正バスの定量比較——ASP3・NuttXの操作回数・値とも一致（唯一の差はI2C_SAR_ADCのポーリング回数のみで返り値は同一）。
- **実施78**：`wifi_regsnap`計装でAGC/CCA層が動的に生きていることを確認（agc_spot/agc_sum が連続的に変化）。MAC割込みは約140/sで発火し続けるが`lmacProcessRxSucData`は0のまま——「部分的に反応」，実施58の局在化を独立チャネルで再確認。
- **実施79→82**：TX電波放射確認（78ラウンドを通じて未検証だった「TXは本当に電波として出ているか」）。promiscuousスニファをボードCへ一時的に書き込む必要があり，権限システムとの間で認可上の問題が発生した（詳細は割愛，事後に訂正・是正済み，最終的にユーザー本人の直接確認を得て実行）。
- **実施80（RX側の決定的所見）**：`lmacRxDone`——`lmacProcessRxSucData`よりさらに上流のROM常駐RX_DONE指標（固定アドレス`0x40000c50`，firmware非依存）。**ASP3は2×60秒ブートで0ヒット，NuttXは60秒で34ヒット**（同一手法で陽性対照確認済み）——正当な802.11 RX_DONEがASP3のMAC/BBハードウェアから一度も発火しないことが，80ラウンドで最も精密なRX局在化として確定。
- **実施81（TX側の決定的所見）**：promiscuousスニファで271件の遠方プローブ（RSSI -83〜-97dBm）を検出できる健全な計装の下で，ボードB由来と判定できるプローブは0件——**RSSIの観点からも「至近距離のはずの信号が全く見えていない」**。TXも実際には電波として放射されていない可能性。
- **実施83（物理的RF信号源投入・決定的）**：ボードC上に既知・確実に強い至近距離ビーコン（SSID固定，signal=94—2位のAPを圧倒）を用意し，ボードBがこれに一切反応しないことを2回の独立起動で確認（`lmacRxDone`/`lmacProcessRxSucData`/`ppRxPkt`/`sta_rx_cb`全て0，MACイベントbit14も0）。**これにより「信号強度が不十分だった可能性」という最後の疑問も解消**——ASP3のRX解析経路は信号強度・距離・環境要因とは無関係に構造的に機能していないと確定。
- **実施84（Direct Boot A/Bテスト）**：標準ESP-IDFの第2段階ブートローダ経由での起動を試す実験を依頼したところ，**過去のラウンド（実施6/21/22/23/30/34）で既に実質的に検証済み**だったと判明：
  - 実際のブートローダ経由起動は実施21で実行済み——機構は機能しASP3はWiFi初期化まで到達したが，実施22で**ハードウェアロック済みPMP/PMA**（ソフトウェアからクリア不可能）による新規クラッシュに阻まれた（技術的ブロックであって反証ではない，AGC/RXはこの経路では一度も観測できていない）。
  - **NuttX自体が`CONFIG_ESPRESSIF_SIMPLE_BOOT=y`でビルドされている**（独立したブートローダバイナリを使わない）——動作する参照実装自体がブートローダを経由していないため，「本物のブートローダが必要か」という問い自体が的外れだった。
  - 未移植の初期化ステップ移植：WiFi関連候補は全て解決済み（`esp_perip_clk_init()`のクロック選択は実施6で修正済み実バグ；`pmu_init()`の全効果は実施30で1KB全体をバイト単位比較し呼出し有無に関わらず同一と実測済み；`esp_clk_init()`と`bootloader_init()`のregi2c 1行は実施34で反証済み）。

# 現在の到達点：3つの主要な調査軸が全て出尽くした

- **値レベルの比較**（静的・動的，実施1-77）：出尽くした。
- **タイミング/物理的RF刺激**（実施78-83）：TX・RXとも「MACレベルではデジタル的に生きているが，RF/アナログ境界を実際に越えた証拠がない」という対称的な結論に達し，物理的な既知強信号にも一切反応しないことまで確認済み。
- **Direct Bootの前提状態**（実施84）：既に検証済み，または技術的に検証不能（PMP/PMAロック）と判明。

残る選択肢：
(a) レジスタ経由では検証しようがない内部アナログ特性（インピーダンス整合・LNAゲイン・ミキサー位相等）
(b) 低確度だが真に未検証な残存選択肢：`esp_rtc_init`+`esp_clk_init`の加算的移植（ハングリスク監視付き），またはPMP/PMAロックを持たないカスタムブートローダの構築（実施22の選択肢iii）
