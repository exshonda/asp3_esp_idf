| 種別 | 実出現 | ユニーク |
|---|---|---|

**認証情報の混入は 0 件**＝ビルド注入のみという規律が実際に守られていた。

## 2. ★機微度の分類（一律に扱わない）

| MAC | 実体 | 機微度 |
|---|---|---|
| `<MAC-38>` | **スマホの identity(public) アドレス**（BLE bond 調査で RPA を解決した結果） | **高**——BLE privacy が隠すはずの永続識別子で、個人の端末に紐づく |
| `<MAC-33>` | **開発PCの Bluetooth アダプタ（`hci0`）** | 中 |
| その他（C3/C5/C6/S3/P4 等） | 自前の開発ボード | 低（電波に出ている値） |

## 3. 置換方式＝**擬似化（追跡可能性を保つ）**

単純な伏字にしない。**このプロジェクトは「どのボードで測ったか」を追える事が規律の中核**
（「board 固有の問題」と誤判定した事故が実際にあり、台帳照合を鉄則にしている）。
一律 `<MAC>` に潰すと**同一ボードかどうかが判別できなくなり、証跡の価値が落ちる**。

⇒ **一意 MAC → `<MAC-NN>` の安定マッピング**（同じボードは常に同じタグ）。
復元表は**リポジトリ外**の私設台帳 `~/usb_mac_pseudonym_map.txt`（600）に退避し、公開しない。

## 4. 置換で機能を壊さないための事前分類（実測）

| 出現箇所 | 性質 | 扱い |
|---|---|---|
| `.md`（58ファイル） | 記録・証跡 | そのまま擬似化 |
| `.c` / `.cmake`（3ファイル） | **すべてコメント**（実験したボードの記録） | 擬似化（機能影響なし。ビルドで確認） |
| `.sh`（10ファイル） | **機能的**——`BOARD_MAC="${BOARD_MAC:-<実MAC>}"` の既定値 | §5 参照 |

## 5. `.sh` の既定値は「env 必須」へ（安全側の副次効果）

擬似化すると既定値が無効な文字列になり helper が壊れる。単に戻すのではなく
**`${BOARD_MAC:?…}` にして未指定なら即エラー**とした（8ファイル）。
これは**「黙って別のボードを触る」事故の防止にもなる**——本プロジェクトでは
過去に、古い記憶のポート番号を信じて**別ポートに電源操作を打ち、しかも出力を捨てて
気づかなかった**事故がある。既定値が無いほうが安全。
未指定で即停止することを実測確認済み。

## 6. 検証

- 残存：**MAC 0 件 / `/home/honda` 0 件**（追跡ファイル・submodule 除く）
- `.cmake` に残る `$HOME` 出現は**すべてコメント**（過去の状態の説明）＝機能に影響なし

## 8. ★履歴の扱い＝**書き換えない（案A）で確定**（ユーザー判断・2026-07-21）

作業ツリーのスクラブは**将来の閲覧者にのみ有効**で、**既に公開された履歴には残っている**
（`git log -p` で到達可能）。この扱いを検討し、**書き換えない**と決定した。

### 8.1 判断材料

| 案 | 効果 | コスト |
|---|---|---|
| **A（採用）** 履歴はそのまま | 今後の閲覧者が見る「現在のファイル」は綺麗 | 履歴を掘れば見える |
| B `git filter-repo` ＋ force push | 履歴からも除去 | ①**全 commit SHA が変わり、docs が引用する実在 SHA 86件が全滅**（本リポジトリは証跡が SHA 引用で成立している）／②既存クローン・フォークが壊れる／③**GitHub 側に古いオブジェクトが残る**（サポートへ削除依頼しない限り到達可能）／④490 commit の書換え |

### 8.2 採用理由

- **B は「完全な除去」にならない**——GitHub の到達可能オブジェクト・既存フォーク・
  外部アーカイブに残る。**目的を達成できないのにコストだけ確実に払う**。
- 本リポジトリ特有のコストが大きい：**証跡が commit SHA の相互参照で成り立っている**
  （実在 SHA の引用が 86件）。書き換えると**過去の実測記録を辿れなくなる**＝
  このプロジェクトが最も大事にしている「後から検証できること」を損なう。
- 露出の中身は**大半が自前の開発ボードの MAC**＝電波に出ている値であり秘密ではない。

### 8.3 残る1件への対処（履歴書換えより確実な代替）

**スマホの identity(public) アドレス**（§2 の「高」）だけは性質が異なる。
これは**端末側で Bluetooth の identity/IRK をリセット**すれば、公開済みの値は
**無効な過去の識別子**になる（Android/iOS とも Bluetooth 設定のリセットで再生成）。
履歴書換えと違い**確実で副作用がない**。実施はユーザーの判断（本ラウンドでは未実施）。

### 8.4 今後の再発防止

- 新規に書く証跡では**最初から `<MAC-NN>` 等の擬似化で書く**
  （復元表はリポジトリ外の私設台帳に置く）。
- `.sh` 等に**実 MAC を既定値として持たせない**（§5 のとおり env 必須にする）。
