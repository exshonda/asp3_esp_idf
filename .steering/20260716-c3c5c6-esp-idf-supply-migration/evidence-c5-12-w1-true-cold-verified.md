# evidence-c5-12 — ★C5 の W1 を «真cold» で検証（README の最後の未検証項目を解消）

2026-07-21。DUT＝C5 `D0:CF:13:F0:C8:94`（hub 1-6 port2+3）。
事前登録＝測定前に予測を記録（本書 §2）。

## 0. 何が未証明だったか

C5 の W1（Wi-Fi DHCP で IP 取得＋ping）は実機で成立していたが、
**それを走らせたブートが POR 起動だったことが未証明**だった。
README「既知の制限」の記述：

> 記録されたブートはいずれも `rst:0x15`（＝POR起動そのものではない）で、
> 採取ハーネス（`rts_boot_capture.py`）が観測対象の cold boot を中断していた。

＝**測る行為が観測対象を壊していた**。C6 の真cold ハングが「warm しか見て
いなかった」ことが真因だった型なので、ここは「達成」と書けなかった。

## 1. ★測定設計上の壁（これを外すと今回も無効になる）

1. **`rst:0x1 (POWERON)` は真cold の証明にならない**。ESP32 は EN リセットと
   電源投入が同一 signature。CP2102N を `dtr=False/rts=False` で open すると
   EN が駆動され、**電源を切っていないのに `rst:0x1` が出る**（C6 で実証済み）。
   ⇒ **signature を根拠にしない。**
2. **CP2102N は DUT と同じ hub ポート系統から給電**されるので、電源断中は USB が
   消える。復電後の再列挙（数秒）を待つ間に ESP32 は起動を終える。
   ⇒ **POR ブートの «先頭» はそもそも録画できない。**

∴ **signature でも «先頭からの録画» でもなく、«状態の遷移» を証拠にする。**

## 2. 事前登録した予測（測定前に記録）

`evidence-c5-09 §8.4(c1)` が同一個体で確立した型を使う：
**LP_AON STORE3（`0x600B100C`）は真 POR で消える**（`0xDEADBEEF`→`0x00000000`）。

| # | 予測 | 外れた場合 |
|---|---|---|
| **P1** | センチネルが読み戻せる | 書込み経路が壊れている＝実験不成立 |
| **P2** | 電源断中に by-id が消える | 電源が切れていない |
| **P3** | **リセットを撃たずに** IP 取得＋ping が観測できる | W1 が POR 起動で成立しない（新事実） |
| **P4** | W1 観測 **後** に STORE3 が `0x00000000` | 真 POR ではなかった ⇒ **主張を取り下げる** |

**4つ全部が揃って初めて「真cold で達成」と書く**と事前に決めた。

## 3. ★交絡の事前潰し

- **アプリが STORE3 を書かないこと**を測定前に確認：本 W1 ビルドの ELF を
  逆アセンブルして `0x600B100C` への参照 **0件**（＝センチネル消失を
  アプリ由来と取り違える経路が無い）。
- 手順5では **esptool を一切使わない**（`--before usb-reset` は DUT をリセット＝
  観測対象のブートを壊す）。読み出しは **W1 観測後**に限定。
- ping の成否は**ログ本文**で判定し `rst:` 行に依存しない。

## 4. 結果（★P1〜P4 全て成立・独立2回）

ビルド＝`apps/wifi_dhcp`・console=uart0（既定）・SSID はビルド時注入
（`strings` で ELF 内に1件確認）。

| 手順 | 1回目 | 2回目 |
|---|---|---|
| **P1** センチネル設置 | `0x600b100c = 0xdeadbeef` ✓ | 再設置 ✓ |
| **P2** 電源断中の by-id | JTAG=0件 / CP2102N=0件 ✓ | 0件 ✓ |
| **P3** リセット無しで W1 | `ping gateway -> OK` **54回** ✓ | `IP acquired: 192.168.1.79`＋**60回** ✓ |
| **P4** W1後の STORE3 | **`0x00000000`** ✓ | **`0x00000000`** ✓ |

観測は `dtr=True/rts=True` で open（**この設定では EN を駆動しない＝DUT を
リセットしない**。C5 で発見・C6 で再測確認済みの性質）。
∴ **観測した W1 は、電源断→復電後に一度もリセットを受けていないブート**のもの。

## 5. 結論

**C5 の W1 は真cold（POR）で達成。** 論証の鎖：

1. センチネルが電源断前に存在した（P1）
2. 電源が実際に切れた（P2＝by-id 消滅の読み戻し）
3. 復電後、**リセットを一切撃たずに** W1 が成立した（P3）
4. その電源断は LP_AON を消していた＝**真 POR だった**（P4）
   ＋アプリは STORE3 を書かない（§3）ので消したのは POR 以外にあり得ない

⇒ 3 で観測した W1 は 4 が示す POR 起動のもの。**独立2回で再現。**
