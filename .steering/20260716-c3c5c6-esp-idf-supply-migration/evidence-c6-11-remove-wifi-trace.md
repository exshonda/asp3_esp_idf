## 5. ★★指摘より «実害が広い» 発見 — **app 側にも同じ反パターンがあり、しかも live なレジスタ書込み**

**`apps/wifi_scan/wifi_scan.c` は `#ifdef TOPPERS_ESP32C6_WIFI`（＝«機能» マクロ）配下で
診断を直接呼んでいる**（＝レビューが指摘したのと**同じ反パターン**が app 側にもある）。
機械的にブロックを列挙して分類した（実測）：

| 行 | 内容 |
|---|---|
| 15-17 | `#include "wifi_trace.h"` |
| 508-549 | DIAGNOSTIC：RTC RAM への osi カウンタ累積 ＋ **`tslp_tsk(300000)`（300ms 待ち）** |
| **553-558** | `wifi_trace_reset()`／`wifi_regi2c_reset()`／**`wifi_regi2c_patch_install()`**／`wifi_taskdelay_reset()` |
| 722-725・759-764 | DIAG syslog |
| **801-808** | `wifi_regsnap_reset()` ＋ **`*(volatile uint32_t *)0x600af018U = 0x7U;`**（「★根本原因テスト（追記10）」） |
| **822-831** | **スキャンループ内で «毎秒» `*(volatile uint32_t *)0x600af018U = 0x7U;`**（「追記10：クロック再アサート」） |

### 5.1 ★なぜ «触らなかった» か（＝ユーザー判断へ返す理由）

1. **`wifi_regi2c_patch_install()` は ROM の `g_phyFuns` テーブルを書き換える**
   （`tbl[WIFI_PHYFUNS_IDX_I2C_WRITE] = wifi_regi2c_traced_write`）＝
   **PHY の regi2c write を «全て» トレース関数へ迂回させる**＝**明白に挙動を変える**。
2. **★`0x600af018 = 0x7` は «診断» ではなく «恒久修正の代行» の可能性が高い**：
   このアドレスは **`MODEM_LPCON_CLK_CONF`**＝**私が `evidence-c6-04` で扱い、
   `bt/bt_shim.c` が `|= 0x1`(WIFIPWR) を «恒久修正» として入れている register と同一**。
   追記10 のコメント自身が「**これでAPが検出できれば，shim のクロック enable 欠落が
   0 AP の根因と確定**」と書いており、**その «テスト» が通ったまま残っている**
   ⇒ **外すと WiFi が 0 AP に戻る可能性がある**（＝**診断の皮を被った修正**）。
3. ∴ **«消す» のは «WiFi のクロック初期化を shim 側へ恒久化する» という別作業**であり、
   **本ラウンドの «計装除去» の射程外**。**壊しうる変更を無断でしない**（親指示）。

### 5.2 ★推奨（ユーザー判断）

**次の1ラウンド**として：**`0x600af018` の設定を app から `wifi/esp_wifi_adapter.c`（shim）側へ
«恒久修正として» 移し**（＝BT が `bt_shim.c` でやっているのと同じ形）、
**その上で app の診断ブロックを `TOPPERS_ESP32C6_WIFI_TRACE` ガードへ付け替える**。
**判別は安い**：移設後に **真cold で `wifi_scan` が AP を取れるか**（今日の 13 APs が基準）。

## 6. ★§12 への含意（依頼 (e)）

**言える**：**§12 の «同一入力で WiFi は収束・BT は非収束» という比較は、
«計装された WiFi»（`--wrap` 37本＋ROM `g_phyFuns` パッチ＋`0x600af018=0x7` の毎秒書込み）と
«無計装の BT» の比較だった**——**これは事実**（本ラウンドの nm/ソース実測）。

**言えない**：**「§12 の結論は計装のせいで誤りだった」**。
- §12 の結論（発散点は `register_chipv7_phy` 内部）を覆すには、**BT 側に同じ計装を入れる**か
  **WiFi から外して同じ比較をやり直す**必要がある＝**本ラウンドでは測っていない**。
- ★ただし **`evidence-c6-05` が §12 の «前提» を既に無効化している**：
  **hal・v5.5.4・v6.1 の3供給すべてが D-1 に到達**＝**«BT が非収束する» という現象自体が
  この個体で再現しない**。∴ **§12 の比較は «再現しない現象» についてのものになっている**。
- ⇒ **§12 は «計装の非対称» と «前提の非再現» の二重で信頼度が下がっている**が、
  **どちらも «§12 が誤りだった» の証明ではない**。**申し送りに留める。**
